# `drizzle` — Generated migrations

SQL migrations generated by `drizzle-kit` from `src/db/schema`.

## Rules

- **Generated, not hand-edited.** Change the schema in `src/db/schema`, then
  regenerate.
- Commit migrations to version control.
- `drizzle.config.ts` (top level) points `drizzle-kit` at the schema and this
  output folder, and always uses `DATABASE_URL` — never `PROD_DATABASE_URL`.
- `meta/` is drizzle-kit's own journal. Generated; leave it alone.

## Workflow

```bash
npm run db:generate   # schema change → new SQL migration here
npm run db:migrate    # apply pending migrations
```

On `main`, CI applies pending migrations against production (ADR-012 —
see `docs/deployment.md`).
